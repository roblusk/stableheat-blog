## How content is organized

Posts come from two sources:

1. **Editorial / topic posts** (26) — guides, comparisons, and explainers that
   aren't tied to a specific customer project. Live as flat `.md` files in
   `src/content/posts/`.

2. **Customer project posts** (48) — drawn from the StableHeat Project
   Catalog (`StableHeat_Project_Catalog.xlsx`). Each is co-located with its
   hero photo as a folder: `src/content/posts/<slug>/{index.md, cover.jpg}`.

## How to add a new post (workflow recap)

1. Find a target in the "Not yet covered" list above
2. Browse the catalog folder path for photos (and optionally Word/PDF docs
   for additional context — read only, don't edit)
3. Create `src/content/posts/<slug>/` with a `cover.jpg` and `index.md`
4. Frontmatter must include: `title, deck, category, date, readTime,
   thumbVariant, thumbIcon, excerpt, applications, heroImage, heroImageAlt`
5. Tag `applications:` with the right slugs from `src/data/applications.ts`
6. Run `npm run build` to verify (and to optimize the photo into multiple WebP variants)
7. Commit and push — GitHub Actions deploys automatically

See [DESIGN.md](DESIGN.md) for voice, tone, and visual guidelines, and the
main [README.md](README.md) for the technical setup.
